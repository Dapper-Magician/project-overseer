# Project Overseer Product Vision

## Product Summary

Project Overseer is a local-first, markdown-powered, AI-native command workstation where President Horne works with a specialized THINKTANK of Directors to turn ideas into artifacts, prototypes, documents, strategies, and ventures.

The product combines:

- multi-agent collaboration
- command workstation UX
- persistent artifacts
- markdown-first documentation
- Director-specific rooms
- ARES tool protocols
- project memory
- design-forward interface
- Codex-assisted implementation workflows

The first version should prove the experience by making the real project canon navigable, beautiful, and operational.

## Primary User

The primary user is President Horne: a prompt engineer, systems thinker, persona architect, builder, and creative strategist who wants a powerful AI command environment for developing projects from ideation to implementation.

The product should support:

- brainstorming
- architecture planning
- prompt engineering
- software prototyping
- documentation
- visual design
- research
- project management
- business development
- content creation
- personal operating systems

## Core Product Loop

The first working product loop is:

1. President Horne enters the Command Bridge.
2. A mission or task is given.
3. Overseer selects or addresses the relevant Director.
4. The Director responds in their own voice.
5. The Director creates or updates an artifact.
6. The artifact is stored in the Artifact Vault.
7. The decision or action is logged in Memory Loom.
8. The user can inspect, copy, revise, or export the result.

## First Build Objective

The first implementation should create a navigable workstation shell with strong visual identity and real repository-backed content.

It should include:

- Command Bridge
- Council Chamber
- Director Rooms
- ARES Armory
- Artifact Vault
- Memory Loom

The first build should render or summarize actual files from this repository, including:

- `docs/`
- `directors/`
- `ares/`
- `prompts/`
- `diagrams/`

## Core Screens

### Command Bridge

The home base.

Shows:

- active mission
- current mode
- Overseer status
- quick navigation
- Director status cards based on `directors/`
- recent artifact cards based on real project files
- recent memory entries based on `docs/meta/`
- command input strip

### Council Chamber

The shared conversation space.

Shows:

- current development focus
- participating Directors
- relevant decisions
- current prompt or task
- artifacts under discussion

### Director Rooms

Each Director has a room.

Shows:

- Director identity from `directors/`
- department
- current focus
- preferred tools
- recent artifacts connected to that Director
- recent notes
- voice and behavioral traits

### ARES Armory

The protocol and tool area.

Shows:

- available ARES protocol files from `ares/`
- REPO
- Kanban
- Memory
- future protocols as they are created

ARES should be presented as an armory/protocol system, not as a person.

### Artifact Vault

The persistent output library.

Shows actual project files such as:

- markdown documents
- diagrams
- generated specs
- prompt files
- Director profiles
- ARES protocols

### Memory Loom

The continuity system.

Shows actual files such as:

- decision log
- session summaries
- open questions
- estimate calibration
- artifact index
- project state notes

## Product Feel

The user should feel like they are commanding a specialized AI company from a futuristic operations console.

The interface should feel powerful, serious, intelligent, and alive.

Avoid the emotional tone of a generic productivity tool.

This is not project management software with AI.

This is an AI command institution that produces work.

## MVP Definition

The first serious MVP succeeds when the user can:

- navigate the core rooms
- inspect Director identities from real Director files
- view ARES tool protocols from real ARES files
- view actual project files in Artifact Vault
- view actual meta-memory files in Memory Loom
- understand the product concept instantly
- feel the difference between Project Overseer and ordinary AI dashboards

The first functional MVP succeeds when the user can:

- create an artifact
- save it
- view it in the Artifact Vault
- log the action or decision
- use a Director-specific profile to shape output
- optionally use an ARES protocol to improve structure
